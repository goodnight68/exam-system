# 智能在线考试系统 V2.0

> 基于移动终端的智能在线考试系统。学生手机扫码即可参加考试，教师在线管理题库与试卷，支持自动评分、错题本、防切屏、成绩导出等功能。

---

## 一、系统简介

本系统采用 B/S 架构，教师和学生均通过浏览器访问，无需安装 APP。系统设计了三种角色：

| 角色 | 能做什么 |
|------|----------|
| **管理员** | 管理所有用户，管理所有试卷和题库，发布系统公告，查看全系统统计 |
| **教师** | 创建和管理题库，设计试卷并发布考试，查看学生成绩与排名，批改简答题，导出成绩 |
| **学生** | 手机登录参加考试，随机练习自测，查看错题本与成绩排名 |

### 核心功能

- 支持五种题型：单选题、多选题、判断题、填空题、简答题
- 客观题（单选/多选/判断）提交后自动评分，分数即时显示
- 简答题由教师在后台手动批改，批改后自动重算总分
- 考试倒计时，超时自动交卷
- 防切屏机制：考试期间切换标签页/窗口超过 3 次，强制交卷
- 答案实时保存，刷新页面或断网重连后可继续答题
- 错题本：交卷时自动收录错题，学生可标记"已掌握"分类复习
- 成绩导出：教师可将学生成绩一键导出为 CSV 文件（Excel 可打开）
- 题目批量导入：按固定格式粘贴文本即可批量创建题目
- 5 套预置试卷 + 80+ 预置题目，开箱即用

## 二、环境准备

### 2.1 需要安装的软件

| 软件 | 版本要求 | 下载地址 |
|------|----------|----------|
| Node.js | v24 或更高 | https://nodejs.org （下载 LTS 版本） |
| MySQL | 8.0 或更高 | https://dev.mysql.com/downloads/mysql/ |

> 提示：安装 MySQL 时请记好 root 密码，后续配置要用。

### 2.2 检查安装是否成功

Windows 下打开命令提示符（Win+R 输入 cmd），分别输入以下命令，有版本号输出即表示安装成功：

```bash
node -v        # 应输出 v24.x.x
npm -v         # 应输出 10.x.x
mysql --version  # 应输出 8.0.x
```

## 三、部署步骤

### 3.1 下载项目

```bash
git clone https://github.com/goodnight68/exam-system.git
cd exam-system
```

或直接在 GitHub 页面点击"Code → Download ZIP"下载解压。

### 3.2 导入数据库（含测试数据）

项目中包含一份带有测试数据的数据库备份文件，一步导入即可拥有账号、题库和试卷。

**方式一：命令行导入（推荐）**

```bash
# Windows 打开命令提示符（cmd），执行：
mysql -u root -p < server\exam_system_dump.sql
# 输入你的 MySQL root 密码
```

**方式二：使用 Navicat / DBeaver 等图形工具导入**

1. 打开 Navicat，连接到 MySQL
2. 右键 → 新建数据库，名称填 `exam_system`，字符集选 `utf8mb4`
3. 右键该数据库 → 运行 SQL 文件 → 选择 `server/exam_system_dump.sql` → 开始

导入成功后会输出：
```
数据库初始化成功
```

### 3.3 配置数据库密码

打开 `server\db.js`，将第 7 行的密码改成你的 MySQL root 密码：

```javascript
const pool = mysql.createPool({
  host: 'localhost',
  port: 3306,
  user: 'root',
  password: '你的密码',   // ← 改这里
  database: 'exam_system',
  waitForConnections: true,
  connectionLimit: 10,
  charset: 'utf8mb4'
});
```

### 3.4 安装依赖

```bash
cd server
npm install
```

看到 `added X packages` 即安装完成。此步骤仅首次需要。

### 3.5 启动服务

```bash
node index.js
```

看到以下输出表示启动成功：
```
智能考试系统 V2.0: http://localhost:3000
监听所有网络接口 (0.0.0.0)，同局域网可访问
MySQL 数据库连接成功
```

### 3.6 访问系统

在浏览器打开 **http://localhost:3000**

如果需要学生在手机上访问，请确保手机和电脑在同一局域网（同一 WiFi），然后用电脑的 IP 地址访问，例如 `http://192.168.1.100:3000`。

> 查看电脑 IP：命令提示符输入 `ipconfig`，找到"无线局域网适配器 WLAN"下的 IPv4 地址。

### 3.7 停止服务

在启动服务的命令行窗口按 `Ctrl + C`，或直接关闭窗口。

## 六、使用指南

### 6.1 教师操作流程

**第一步：注册/登录**

用教师账号登录（如 `teacher / 123456`），进入教师工作台。

**第二步：管理题库**

点击底部"题库"标签，可以：
- 点击"添加题目"逐题录入
- 点击"批量导入"粘贴多题文本（格式见下方说明）
- 支持按分类、题型、难度、关键词筛选题目
- 编辑或删除已有题目

**第三步：创建试卷**

点击底部"试卷"标签 → "创建试卷"：
1. 填写试卷名称、描述、考试时长、及格分数
2. 从题库中勾选题目加入试卷
3. 可选开启"随机抽题"模式
4. 保存后点击"发布"，学生端即可看到

**第四步：查看成绩**

在试卷列表中点击某套试卷的"成绩"，可查看所有参考学生的得分与排名。点击"导出"可将成绩下载为 CSV 文件。

**第五步：批改简答题**

在试卷列表点击"简答批改"，可逐个查看学生答案并手动给分，系统自动重算总分。

### 6.2 题目批量导入格式

每行一道题，字段用 `|` 分隔：

```
题型|难度|题目内容|答案|解析|分值
```

示例：
```
单选|简单|HTTP协议默认端口是？|80|HTTP默认80端口,HTTPS为443|5
多选|中等|TCP协议的特点包括？|面向连接,可靠传输,流量控制,拥塞控制|UDP速度快但不可靠|6
判断|简单|Python是解释型语言。|对|Python逐行解释执行|5
填空|中等|HTTP状态码404表示____。|资源未找到|404 Not Found|6
简答|困难|简述数据库事务的ACID特性。|原子性:...一致性:...隔离性:...持久性:...|ACID是事务核心|8
```

> 注意：多选答案用英文逗号分隔；填空答案可以有多个，用中文逗号分隔。

### 6.3 学生操作流程

1. 用学生账号登录（如 `student / 123456`）
2. 首页查看已发布的考试，点击进入考试
3. 答题时切换题目会自动保存答案
4. 答完点击"交卷"，查看得分与答题详情
5. 错题自动收入"错题本"，可标记已掌握
6. 可在首页进入"随机练习"模式自测

## 八、常见问题

**Q：MySQL 连接失败怎么办？**
A：确认 MySQL 服务已启动（Win+R → services.msc → 找到 MySQL80 → 启动），确认 `db.js` 中的密码正确。

**Q：端口 3000 被占用怎么办？**
A：修改 `server/index.js` 第 16 行的端口号，例如改成 `3001`。

**Q：学生手机怎么访问？**
A：确保手机和电脑在同一 WiFi，用 `ipconfig` 查电脑 IP，手机浏览器访问 `http://你的IP:3000`。

**Q：如何重置所有数据？**
A：重新导入数据库备份文件即可：
```bash
mysql -u root -p < server\exam_system_dump.sql
```

**Q：题目中的图片怎么上传？**
A：教师添加题目时可点击上传图片按钮，图片保存在 `server/uploads/` 目录。
