# 智能在线考试系统设计与实现

## 摘 要

本系统采用前后端分离架构，前端使用HTML5、CSS3、JavaScript开发移动端自适应单页面应用，后端基于Node.js+Express搭建服务接口，采用MySQL数据库完成全部业务数据持久化存储。

**关键词：** 移动开发；在线考试；H5；Node.js；MySQL；前后端分离；错题本

---

## 一、引言

### 1.1 背景与意义

随着信息技术的迅猛发展，教育领域正经历着深刻的数字化转型。传统的纸质考试模式存在组织成本高、阅卷效率低、统计分析困难等问题。移动终端由于其便携性和普及率，已成为在线教育的重要载体。开发一款智能在线考试系统，能够有效提升考试效率，降低教学管理成本，并为学生提供更加灵活便捷的考试体验。

### 1.2 主要工作

本文完成的主要工作包括：
1. 基于 HTML5 技术构建移动端自适应前端界面，采用 SPA 单页面架构
2. 使用 Node.js 和 Express 框架搭建后端 RESTful API 服务，设计 30+ 接口
3. 采用 MySQL 关系型数据库，设计 11 张数据表，使用事务保证数据一致性
4. 实现三角色（学生/教师/管理员）权限体系，JWT + bcrypt 认证方案
5. 实现多题型在线考试、客观题实时自动评分、主观题教师批改功能
6. 实现防切屏检测机制（基于 Page Visibility 和 blur 事件）
7. 实现错题本模块，自动收录考试错题并支持复习标记
8. 实现公告系统、知识点管理、全系统管理后台等辅助功能

## 二、系统需求分析

### 2.1 功能需求

本系统设计三种用户角色：管理员（Admin）、教师（Teacher）和学生（Student），各角色的功能需求如下：

**学生端功能：**
- 用户注册与登录（支持学号、院系等扩展信息）
- 首页公告查看（教师/管理员发布的系统通知）
- 查看可参加的考试列表
- 随机练习模式（从全题库随机抽取题目自测）
- 在线答题（支持单选、多选、判断、填空、简答题型）
- 考试计时与超时自动交卷
- 防切屏检测（检测页面隐藏/窗口失焦，超3次强制交卷）
- 答案实时保存（每道题切换即提交到服务端）
- 客观题自动评分，交卷后即时显示得分
- 查看考试成绩与答题详情（含正确答案与解析对照）
- 考试记录与历史成绩查看
- 考试排名查看（基于 ROW_NUMBER 窗口函数）
- 错题本（自动收录错题，支持标记已掌握、删除）
- 个人信息管理（昵称、头像、性别、生日、邮箱等）

**教师端功能：**
- 题库管理（题目的增删改查，支持分类、题型、难度、关键词多条件筛选）
- 题目批量导入（按固定格式粘贴文本，批量创建题目）
- 试卷管理（创建/编辑/删除试卷，支持随机抽题模式）
- 发布/关闭试卷
- 成绩管理（查看各场考试的学生成绩排名）
- 成绩 CSV 导出（含 BOM 头兼容 Excel 中文）
- 每题正确率统计（含正确人数/总答题数/正确率百分比）
- 简答题批改（查看学生答案并手动评分）
- 错题本统计（按分类统计学生错题分布）
- 数据统计面板（题目数量、考试人次、平均分等）
- 公告发布（面向全体学生的系统通知）
- 个人信息管理

**管理员端功能：**
- 用户管理（创建/编辑/删除用户，重置密码，启用/禁用账号）
- 全系统试卷管理（查看/删除任意教师的试卷）
- 全系统题库管理（查看/删除任意教师的题目）
- 公告管理（发布/编辑/删除/置顶公告）
- 知识点管理（创建/编辑/删除知识点标签）
- 成绩总览（查看所有考试记录与得分）
- 全系统统计（总用户数、总题目数、总试卷数、平均分、角色分布）

### 2.2 非功能需求

- **界面要求**：界面设计美观友好，具备移动端操作体验，限制最大宽度模拟原生应用
- **数据安全**：用户密码 bcrypt 加盐哈希存储，JWT Token 认证机制保障接口安全，三级权限校验（authMiddleware / teacherOnly / adminOnly）
- **数据一致性**：试卷创建与题目关联使用数据库事务（beginTransaction/commit/rollback）保证原子性
- **防作弊**：考试页面实施防切屏监控，同时检测页面隐藏和窗口失焦事件，超过阈值自动交卷
- **数据持久化**：所有业务数据存储于 MySQL 8.0 关系型数据库，使用 utf8mb4 字符集支持特殊字符

## 三、系统设计

### 3.1 技术架构

系统采用前后端分离的 B/S 架构，整体分为三层：

1. **表现层（前端）**：HTML5 + CSS3 + JavaScript，单页面应用（SPA），移动端自适应布局
2. **业务逻辑层（后端）**：Node.js + Express 框架，提供 RESTful API
3. **数据持久层**：MySQL 8.0 关系型数据库

```
┌─────────────────────────────────────┐
│           移动端浏览器 (H5)           │
│       HTML5 + CSS3 + JavaScript      │
│           SPA 单页面应用              │
└──────────────┬──────────────────────┘
               │ HTTP/JSON
┌──────────────▼──────────────────────┐
│          Node.js + Express           │
│       RESTful API 接口服务            │
│     JWT 身份认证 & 业务逻辑处理       │
└──────────────┬──────────────────────┘
               │ mysql2
┌──────────────▼──────────────────────┐
│           MySQL 8.0 数据库            │
│      exam_system 数据持久化存储        │
└─────────────────────────────────────┘
```

### 3.3 API 接口设计

系统遵循 RESTful 风格设计 API，采用 JSON 格式通信，使用 JWT (Bearer Token) 进行身份认证。API 统一返回格式为 `{ code: 200/400/500, msg, data }`。主要接口如下：

**用户模块（/api/user）：**
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /register | 用户注册（支持学号、院系等） | 公开 |
| POST | /login | 用户登录，返回 JWT Token | 公开 |
| GET | /info | 获取当前用户详细信息 | 登录 |
| PUT | /info | 更新个人资料（昵称、邮箱等） | 登录 |

**题库模块（/api/question）：**
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /categories | 获取题目分类列表 | 公开 |
| GET | / | 分页查询题目（支持分类/题型/难度/关键词筛选） | 登录 |
| GET | /all | 获取全部题目简要列表（供教师选题） | 教师 |
| GET | /practice | 随机抽取练习题目（默认20题） | 登录 |
| POST | / | 添加单个题目 | 教师 |
| POST | /import | 批量导入题目（文本格式） | 教师 |
| PUT | /:id | 编辑题目 | 教师 |
| DELETE | /:id | 删除题目（软删除，status=0） | 教师 |

**考试模块（/api/exam）：**
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | / | 创建试卷（含关联题目，事务保证） | 教师 |

### 4.3 用户认证与权限控制

系统采用 `bcryptjs` 实现密码的加盐哈希存储（saltRounds=10），使用 `jsonwebtoken` 实现无状态 Token 认证。认证体系分为三个层级，通过中间件链式调用实现细粒度权限控制：

```javascript
// 密码加密存储（注册时）
const hash = await bcrypt.hash(password, 10);

// Token 生成（登录时，有效期24小时）
const token = jwt.sign(
  { id: user.id, username: user.username, role: user.role },
  JWT_SECRET,
  { expiresIn: '24h' }
);
```

认证中间件 `authMiddleware` 从 HTTP 请求头中提取 `Authorization: Bearer <token>`，使用 `jwt.verify()` 验证并解析用户身份信息，将 `{ id, username, role }` 挂载到 `req.user` 供后续路由使用。`teacherOnly` 中间件校验用户角色是否为 `teacher` 或 `admin`，`adminOnly` 中间件进一步限制为仅 `admin` 角色可访问。前后端均将 Token 存储在 `localStorage` 中，每次 API 请求自动附带，Token 过期或验证失败时前端自动清除登录状态并跳转至登录页。

### 4.4 自动评分实现

系统在两道环节进行评分：提交单个答案时实时判分，以及交卷时汇总得分。针对不同题型采用差异化的评分策略：

**单选题/判断题**：将学生答案与数据库中的正确答案进行字符串匹配（支持大小写不敏感），完全匹配即得分，否则得 0 分。

**多选题**：先将正确答案和学生答案分别转换为大写字母数组，各自排序后拼接为有序字符串再比较。多选答案顺序不影响判定，但必须所有选项完全一致才得分（部分得分不得分）。

**填空/简答题**：不进行自动评分，将答案原样保存，`is_correct` 字段设为 NULL（待批改状态），由教师后续在"简答题批改"页面手动评分。批改后系统自动重新计算该考试记录的总分与通过状态。

```javascript
// 提交单个答案时的实时判分逻辑
if (['single', 'judge'].includes(question.type)) {
  isCorrect = (answer === question.answer) ? 1 : 0;
  score = isCorrect ? Number(question.score) : 0;
} else if (question.type === 'multiple') {
  const correct = String(question.answer).toUpperCase().split('').sort().join('');
  const studentAnswer = String(answer).toUpperCase().split('').sort().join('');
  isCorrect = (correct === studentAnswer) ? 1 : 0;
  score = isCorrect ? Number(question.score) : 0;
}
```

交卷时，后端执行 `SELECT SUM(score) FROM exam_answers WHERE record_id=?` 汇总所有题目得分，与试卷的 `pass_score` 比较判定是否及格，同时将回答错误的题目自动插入 `wrong_book` 错题本表，使用 `INSERT IGNORE` 防止重复收录。交卷完成后考试记录状态变为 `graded`。

### 4.5 数据库事务处理

试卷创建和编辑操作涉及多表联动（exams 表 + exam_questions 表），为保证数据一致性，系统使用 MySQL 事务机制。以创建试卷为例：先从连接池获取连接、开启事务，依次插入试卷记录和题目关联记录，全部成功后 `commit()`；任一环节失败则 `rollback()` 回滚所有操作，最后在 `finally` 块中释放连接回连接池。

```javascript
const conn = await pool.getConnection();
try {
  await conn.beginTransaction();
  const [result] = await conn.query('INSERT INTO exams (...) VALUES (...)');
  const examId = result.insertId;
  if (question_ids && question_ids.length > 0) {
    const values = question_ids.map((qid, i) => [examId, qid, i + 1]);
    await conn.query('INSERT INTO exam_questions (...) VALUES ?', [values]);
  }
  await conn.commit();
} catch (err) {
  await conn.rollback();
  throw err;
} finally {
  conn.release();
}
```

编辑试卷时同样使用事务：先删除旧的题目关联（`DELETE FROM exam_questions WHERE exam_id=?`），再批量插入新关联，并自动汇总题目分值更新 `total_score` 字段。这种"全删全插"策略简化了差异对比逻辑，且事务保证了在删除后插入前不会出现中间不一致状态。

### 4.6 考试计时与自动提交

前端使用 JavaScript `setInterval` 实现每秒更新的考试倒计时。剩余时间初始值为试卷规定的考试时长（分钟），启动考试时后端记录 `start_time` 和 `end_time`（NOW() + 时长分钟），前端以秒为单位逐秒递减。当剩余时间耗尽时，自动调用交卷接口，设置标志位 `examAutoSubmit = true` 以区别于手动交卷：

```javascript
this.examTimer = setInterval(() => {
  this.remainingSeconds--;
  if (this.remainingSeconds <= 0) {
    clearInterval(this.examTimer);
    this.examAutoSubmit = true;
    this.showToast('⚠️ 考试时间到，自动交卷');
    this.doSubmitExam();
    return;
  }
  this.renderTimer();
  if (this.remainingSeconds <= 300) {
    timerEl.classList.add('warning'); // 倒计时5分钟警告
  }
}, 1000);
```

为保证数据可靠性，系统采用"前端计时 + 后端校验"双重机制。当学生离开考试页面后再次进入时，调用开始考试接口会检测是否存在进行中的记录，若有则计算已过时间，若已超时则执行后端自动提交并返回提示；若未超时则恢复剩余时间和已保存的答案，实现断点续考功能。

### 4.7 前端页面路由

系统前端采用单页面应用（SPA）架构，通过控制页面 DOM 元素的 CSS `display` 属性实现页面切换。`APP.showPage(page)` 为核心路由函数，先隐藏所有 `.page` 元素，再激活目标页面。根据 `currentPage` 值自动触发对应数据加载方法。底部导航栏（TabBar）仅在首页/考试记录/错题本/个人中心四个主要页面显示：

```javascript
showPage(page) {
  document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
  document.querySelectorAll('.modal').forEach(m => m.classList.remove('active'));
  const pageEl = document.getElementById('page-' + page);
  if (pageEl) pageEl.classList.add('active');
  this.currentPage = page;
  // 根据页面自动加载对应数据
  const loadMap = {
    home: 'loadHome', questions: 'loadQuestions',
    'exam-manage': 'loadExamManage', records: 'loadMyRecords',
    wrongbook: 'loadWrongBook', profile: 'loadProfile',

```

页面状态不依赖 URL hash，通过全局变量 `APP.currentPage` 追踪，页面切换时调用对应数据加载函数动态渲染内容。这种设计使得系统无需前端路由库即可实现完整的页面导航。

### 4.9 防切屏监控实现

为防止考生在考试过程中切换浏览器标签页或窗口进行作弊，系统实现了前端防切屏检测机制。该机制同时监听两个浏览器事件：

- **Page Visibility API (`visibilitychange`)**：当用户切换标签页、最小化浏览器或锁屏时，`document.hidden` 变为 `true`，触发警告
- **Window blur 事件**：当浏览器窗口失去焦点（如点击桌面其他窗口）时触发

```javascript
let cheatCount = 0;
const warn = () => {
  this.cheatCount++;
  if (this.cheatCount >= 3) {
    this.showToast('⚠️ 检测到多次切屏，已自动交卷！');
    this.doSubmitExam();      // 超过3次强制交卷
  } else {
    this.showToast(`⚠️ 第${this.cheatCount}次切屏警告！超过3次将自动交卷`);
  }
};
document.addEventListener('visibilitychange', () => { if (document.hidden) warn(); });
window.addEventListener('blur', warn);
```

计数器 `cheatCount` 记录切屏次数，第 1-2 次仅弹出警告提示，第 3 次直接调用 `doSubmitExam()` 强制交卷。该机制无需后端配合，完全在前端实现，轻量高效且对正常考试行为无干扰。考试结束或退出页面时需调用 `unbindCheatDetection()` 移除事件监听，避免内存泄漏。

### 4.10 错题本模块实现

错题本是系统的辅助学习功能模块。当学生提交试卷时，后端自动将 `is_correct=0` 的答题记录写入 `wrong_book` 表：

```javascript
// 交卷时自动收录错题
const [wrongAnswers] = await pool.query(
  'SELECT question_id, answer FROM exam_answers WHERE record_id=? AND is_correct=0', [record_id]
);
for (const wa of wrongAnswers) {
  await pool.query(
    'INSERT IGNORE INTO wrong_book (user_id, question_id, exam_record_id, wrong_answer) VALUES (?,?,?,?)',
    [req.user.id, wa.question_id, record_id, wa.answer]
  );
}
```

使用 `INSERT IGNORE` 确保同一道题在同一场考试中只被收录一次（通过联合唯一索引去重）。错题本页面按时间倒序展示错题列表，每道错题关联显示题目内容、正确答案、解析和所属分类。学生可以：
- **标记已掌握**：将错题标记为 `mastered=1`，方便筛选未掌握的题目重点复习
- **删除错题**：从错题本中移除
- **练习模式**：在随机练习中答错的题目也会自动记入错题本，如果同一题多次犯错则增加 `review_count` 复习计数

错题统计接口返回总错题数、已掌握数和按分类分布的错题数量，帮助学生了解知识薄弱点。

### 4.11 管理员后台实现

系统引入第三种角色 admin（管理员），拥有超越教师和学生的最高权限。管理员模块 `routes/admin.js` 包含用户管理、试卷管理、题库管理、公告管理、知识点管理和全系统统计六大功能域，均受 `adminOnly` 中间件严格保护。

管理员可以跨教师查看和操作所有数据，例如查看所有用户列表、删除任意教师的试卷或题目（硬删除）、启用或禁用用户账号等。在删除用户时，系统自动保护 admin 角色不被删除（`WHERE role!=?` 条件过滤），防止误操作导致无法管理系统。

全系统统计接口通过聚合查询返回系统整体运行情况：
```javascript
const [users] = await pool.query('SELECT COUNT(*) as c FROM users');
const [questions] = await pool.query('SELECT COUNT(*) as c FROM questions WHERE status=1');
const [exams] = await pool.query('SELECT COUNT(*) as c FROM exams');
```

### 4.12 成绩排名与导出

排名查询使用 MySQL 8.0 的 `ROW_NUMBER()` 窗口函数，按成绩降序生成连续排名，避免并列排名跳跃问题。同时为学生单独查询其个人排名，通过子查询 `SELECT COUNT(*)+1 ... WHERE score > r.score` 实现。

```sql
SELECT u.nickname, r.score,
  ROW_NUMBER() OVER (ORDER BY r.score DESC) as `rank`
FROM exam_records r JOIN users u ON r.student_id=u.id
WHERE r.exam_id=? AND r.status='graded'
ORDER BY r.score DESC
```

成绩导出功能将查询结果组装为 CSV 格式，在文件头部添加 BOM 字符（`\uFEFF`）确保在 Excel 中正确识别中文编码。设置 HTTP 响应头 `Content-Type: text/csv; charset=utf-8` 和 `Content-Disposition: attachment; filename=scores.csv`，浏览器将自动触发文件下载。

### 4.13 题目批量导入

为提升教师录入效率，系统支持按固定格式的文本批量导入题目。教师只需在文本框中按 `题型|难度|题目内容|答案|解析|分值` 的格式每行一道题贴上，后端逐行解析并插入数据库：

```javascript
const lines = text.split('\n').filter(l => l.trim());
for (const line of lines) {
  const parts = line.split('|');
  if (parts.length < 4) continue;
  await pool.query('INSERT INTO questions (...) VALUES (?,?,?,?,?,?,?,?,?)', [...]);
}
```

每次导入返回成功导入的题目数量，单次最多可导入数十题，效率远高于逐题添加。

### 4.14 答案实时保存机制

考试过程中，学生每切换一道题或点击下一题/上一题时，前端自动调用 `POST /api/exam/answer` 接口保存当前答案。该接口接收 `record_id, question_id, answer` 参数，对客观题执行实时判分后将结果一起写入 `exam_answers` 表。这种设计有以下优势：
- **防止数据丢失**：即使浏览器崩溃或网络中断，已提交的答案已持久化
- **支持断点续考**：重新进入考试时从数据库恢复已答状态
- **后端时间校验**：每次保存答案前验证考试记录是否仍处于 `in_progress` 状态，防止考试结束后继续提交答案

### 6.1 总结

本文设计并实现了一款基于移动终端的智能在线考试系统。系统采用前后端分离架构，前端基于 HTML5/CSS3/JavaScript 打造移动端 H5 单页面应用，后端基于 Node.js + Express 提供 RESTful API 服务，数据采用 MySQL 8.0 关系型数据库进行持久化存储。

系统设计了学生、教师和管理员三种角色，实现了 11 张数据表、30+ API 接口、完整的 CRUD 业务逻辑和三级权限控制体系。核心功能涵盖用户管理、题库管理、试卷管理、在线考试、自动评分与主观题批改、防切屏监控、错题本、公告通知、知识点管理、成绩排名与 CSV 导出、全系统管理后台等模块。支持单选、多选、判断、填空、简答五种考试题型，客观题提交即评分，主观题支持教师手动批改并自动重算总分。考试流程具备倒计时提醒、超时自动交卷、答案实时保存和断点续考等完善机制，防切屏检测通过 Page Visibility API 和 blur 事件双重监听保障考试公平性。

### 6.2 创新点

1. **三角色权限体系**：引入 admin 管理员角色，实现 teacherOnly 和 adminOnly 两级中间件，权限控制细粒度
2. **防切屏监控**：基于浏览器原生 Visibility API + blur 事件，无需后端配合即可实现切屏检测与自动交卷
3. **错题本自动收录**：交卷时自动将错题写入错题本表，支持标记掌握和分类统计，辅助学生针对性复习
4. **事务保证一致性**：试卷创建/编辑使用数据库事务（beginTransaction/commit/rollback），确保多表操作原子性
5. **多题型智能评分**：支持 5 种题型，客观题实时判分，主观题支持教师批改并自动重算总成绩
6. **移动端优化**：界面专为移动端设计，max-width:480px 居中布局，底部固定导航栏，交互接近原生应用
7. **答案实时持久化**：每道题切换即保存，支持断点续考，防止数据丢失

### 6.3 展望

未来可以在以下方面进一步优化：
- 引入随机抽题组卷算法，根据知识点分布和难度系数自动生成试卷
- 集成 AI 辅助评分，对简答题内容进行语义分析自动判分
- 使用 WebSocket 替代轮询，实现教师端实时监考（在考生切屏时即时告警）
- 开发真正的微信小程序版本，利用微信生态降低使用门槛
- 添加数据可视化仪表盘（ECharts 图表展示成绩分布、知识点掌握度等）
- 引入操作日志审计与 IP 归属地记录，提升系统安全审计能力
